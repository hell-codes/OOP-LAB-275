## &nbsp;рҹ“Ӯ &nbsp;Section Breakdown

<br/>

<details>
<summary><b>рҹҺӯ &nbsp;Abstraction</b> &nbsp;вҖ” &nbsp;Separating <i>what</i> an object does from <i>how</i> it does it</summary>

<br/>

| Sub-folder | Focus Area |
|------------|------------|
| `abstract-class` | Defining partially implemented base classes |
| `access-modifiers` | Controlling visibility: `public` В· `private` В· `protected` В· default |
| `implements-multiple` | A class implementing multiple interfaces simultaneously |
| `interface` | Pure abstraction вҖ” declaring contracts without implementation |
| `non-access-modifiers` | Behaviour modifiers: `static` В· `final` В· `abstract` В· `synchronized` |

</details>

<br/>

<details>
<summary><b>рҹ§¬ &nbsp;Inheritance</b> &nbsp;вҖ” &nbsp;Acquiring properties and behaviours from a parent class</summary>

<br/>

- **Single Inheritance** вҖ” One parent, one child
- **Multilevel Inheritance** вҖ” Chain of parent вҶ’ child вҶ’ grandchild
- **Hierarchical Inheritance** вҖ” One parent, multiple children
- **Method Reuse** вҖ” Avoiding code duplication using `extends`

</details>

<br/>

<details>
<summary><b>рҹ”„ &nbsp;Polymorphism</b> &nbsp;вҖ” &nbsp;One name, many behaviours</summary>
